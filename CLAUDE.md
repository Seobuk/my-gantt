# CLAUDE.md — WBS Gantt

프로젝트 통합 일정을 **일(day) 단위 · 3단계 WBS 계층**으로 보고 편집하는 인터랙티브 간트차트.
개인의 다년도 과제 일정을 한 화면에서 관리하기 위한 도구다.

이 문서는 Claude Code가 이 저장소에서 작업할 때 따라야 할 규칙과 구조를 정의한다.

---

## 1. 저장소 구조

```
wbs-gantt/
├── CLAUDE.md                 # 이 문서
├── README.md                 # 빠른 시작
├── web/
│   └── projects_gantt.html   # 단일 파일 웹 앱 (UI + 로직 전부 / 빌드 불필요)
├── data/                     # ★ 데이터 저장소 (raw data가 여기 보관됨)
│   ├── wbs_sample.json        #   시드 데이터 (수정 금지, 초기값)
│   └── wbs.json               #   실제 작업 데이터 (서버가 저장 시 생성/갱신)
└── python/
    ├── app.py                # Flask 백엔드 (HTML 서빙 + 파일 저장 API)
    └── requirements.txt
```

**원칙**: 사용자의 실제 일정 데이터는 항상 `data/` 폴더의 JSON 파일에 둔다.
코드(`web/`, `python/`)와 데이터(`data/`)를 분리한다. `wbs_sample.json`은 건드리지 말고,
변경은 `wbs.json`에만 반영한다.

## 2. 실행 방법

### A) 가장 간단 — 파일 그대로 열기
`web/projects_gantt.html`을 브라우저로 열면 동작한다.
이 경우 저장은 **브라우저 localStorage**(키 `wbs_gantt_v1`)에 된다. 기기/브라우저 종속.

### B) 권장 — Python 서버로 파일 저장
```bash
cd python
pip install -r requirements.txt
python app.py
# http://127.0.0.1:5173 접속
```
서버로 열면 데이터는 **`data/wbs.json` 파일에 저장**된다(여러 기기에서 git으로 공유 가능).
HTML은 다음 순서로 데이터를 처리한다:
1. 시작 시 `GET /api/wbs` 로 서버 데이터를 읽음 → 없으면 localStorage → 없으면 코드 내 기본값
2. 편집 때마다 `POST /api/wbs` 로 서버 저장 + localStorage 동시 백업

> HTML은 **이중 모드**다. 서버가 있으면 파일에, 없으면 localStorage에 저장한다.
> 이 동작(`web/projects_gantt.html`의 `init()`, `saveData()`, `applyData()`)을 깨지 않도록 주의.

## 3. 데이터 모델 (WBS 트리)

`data/*.json`은 노드 객체의 **배열**이다. 노드는 두 종류:

```jsonc
// 상위 노드(프로젝트/서브시스템) — 자식이 있으면 기간·진행률은 자동 롤업
{ "name": "Mobile App", "co": "Product", "col": "#1baf7a",
  "children": [ ... ] }

// 말단 작업(leaf) — 실제 날짜를 가짐
{ "name": "Design", "s": "2026-01-01", "e": "2026-03-31", "pr": 100 }
```

필드:
- `name` (필수): 작업/프로젝트 이름.
- `children` (배열): 있으면 상위 노드. 임의 깊이 가능(3단계 이상도 동작).
- `s`, `e`: 시작일·종료일, **`YYYY-MM-DD` (일 단위)**. `e`는 **그 날 포함**(inclusive).
- `pr`: 진행률 0–100 정수.
- `co`: 수식어/협력사. **최상위 노드에서만** 표시됨.
- `col`: 프로젝트 색(hex). **최상위 노드에서만** 의미 있음. 하위는 최상위 색을 상속.

규칙:
- **롤업**: 상위 노드의 기간 = 하위 leaf들의 (최소 시작 ~ 최대 종료),
  진행률 = 하위 leaf들의 **기간(일수) 가중 평균**.
- **WBS 번호**(1 / 1.1 / 1.1.1)는 트리 위치로 **자동 생성**. 데이터에 저장하지 않는다.
- 좌표 기준 원점은 `2025-01-01`(코드의 `Y0`), 범위 `Y0~Y1`(=2027). 연도 범위를 바꾸려면 `Y0/Y1` 수정.

## 4. 구현된 기능 (web/projects_gantt.html)

- 일 단위 간트 막대, 연도 탭(전체/2025/2026/2027), 실제 오늘 날짜 기준선.
- 3단계+ WBS 트리: 펼치기/접기, 상위 롤업 막대(슬림) vs 말단 작업 막대(라벨 표시).
- 상세 패널 편집: 이름·수식어(연필), 시작일·종료일(date picker), 진행률(slider), 완료/미착수.
- 구조 편집: `+ 형제 추가`, `+ 하위 추가`, `← 상위로/→ 하위로`(indent/outdent),
  `↑/↓`(형제 순서), `이동 위치 ▾`(다른 프로젝트로 트리째 이동), `삭제`.
- 프로젝트 필터 칩, 상단 통계(평균 진행률/진행 중·완료 작업/프로젝트 수).
- 저장: `WBS 업데이트`(즉시 저장+토스트), 자동 저장, `백업 복사`(JSON 클립보드), `초기화`(기본값 복원).
- 라이트/다크 모드 대응. 외부 의존성 없음(순수 HTML/CSS/Vanilla JS).

---

## 5. 코딩 규칙

- `web/projects_gantt.html`은 **단일 파일 · 무빌드 · 순수 vanilla JS(ES5 호환 스타일)**. 프레임워크/번들러 도입 금지.
- 색은 항상 CSS 변수(`--ink`, `--bg`, `--accent` 등)로. 라이트/다크 둘 다 확인.
- 날짜는 절대 month 단위로 되돌리지 말 것 — **day 단위 유지**.
- 데이터 스키마(위 3절)를 바꾸면 `wbs_sample.json`, 롤업/렌더/저장 로직을 함께 갱신.
- Python은 표준 라이브러리 + Flask만. 무거운 의존성 추가 전 확인.
- 간결한 결과물과 반복적 개선을 선호. 큰 변경은 단계적으로.
